## 定长滑动

**定长滑动窗口**（Fixed Length Sliding Window）是滑动窗口算法中最基础、最规范的一种。

它的核心逻辑非常简单：**维护一个长度永远为 `k` 的窗口，右边进一个元素，左边就必须出一个元素**。

相比于“不定长窗口”（求最长/最短子数组），定长窗口通常不需要 `while` 循环来收缩左边界，只需要一个 `if` 判断即可。

以下是为你准备的通用模板，分为 **C++**（你的主力语言）和 **Python**（你的备用语言）。

------

### 1. C++ 通用模板 (推荐)

这个模板采用**单次遍历**的写法，代码最简洁，逻辑最统一。

**核心口诀：一进、二算、三出**

```c++
/*
 * 假设我们要维护一个长度为 k 的窗口
 * 场景：求长度为 k 的子数组的最大和/平均值/特定字符数量等
 */

// 结果变量初始化
double max_sum = -1e9; // 根据题目要求初始化
double current_sum = 0; // 当前窗口的状态

for (int i = 0; i < nums.size(); ++i) {
    // 1. 【进】元素进入窗口
    current_sum += nums[i];

    // 2. 【判断】窗口还没满 k 个时，跳过计算
    if (i < k - 1) {
        continue;
    }

    // --- 此时窗口范围是 [i - k + 1, i]，长度正好是 k ---

    // 3. 【算】更新答案
    max_sum = max(max_sum, current_sum);

    // 4. 【出】最左边的元素离开窗口（为下一次右移做准备）
    // 最左边元素的下标是 i - k + 1
    current_sum -= nums[i - k + 1];
}

return max_sum;
```

#### 💡 针对 AI Infra/C++ 选手的优化 Tip：

如果题目涉及**字符计数**（比如判断异位词），**不要**用 `std::unordered_map`，因为常数时间太大。

- **请直接用数组**：`vector<int> count(26, 0);` 或者 `int count[128] = {0};`
- 这是 C++ 刷题必须养成的性能直觉。

### 2. Python 通用模板

Python 的逻辑完全一样，胜在写起来快。

Python

```c++
def fixedSlidingWindow(nums: List[int], k: int) -> int:
    curr_sum = 0
    ans = float('-inf') # 或者根据题目初始化
    
    for i, x in enumerate(nums):
        # 1. 【进】
        curr_sum += x
        
        # 2. 窗口未满，继续填
        if i < k - 1:
            continue
            
        # 3. 【算】更新答案
        ans = max(ans, curr_sum)
        
        # 4. 【出】移除最左边的元素
        # 最左边元素的索引是 i - k + 1
        curr_sum -= nums[i - k + 1]
        
    return ans
```

### 3. 实战演练：LeetCode 643. 子数组最大平均数 I

**题目**：给一个整数数组 `nums` 和整数 `k`，找出平均数最大的长度为 `k` 的子数组，返回该最大平均数。

**套用 C++ 模板：**

C++

```c++
class Solution {
public:
    double findMaxAverage(vector<int>& nums, int k) {
        double current_sum = 0;
        double max_sum = -DBL_MAX; // 初始化为最小值

        for(int i = 0; i < nums.size(); ++i) {
            // 1. 进
            current_sum += nums[i];

            // 2. 还没满 k 个？跳过
            if(i < k - 1) continue;

            // 3. 算 (因为分母 k 是一样的，比较和就等于比较平均数，最后再除以 k)
            max_sum = max(max_sum, current_sum);

            // 4. 出
            current_sum -= nums[i - k + 1];
        }

        return max_sum / k;
    }
};
```

### 4. 常见变体与应对

定长窗口虽然简单，但有两个常见的变体需要注意：

1. **哈希表/计数器窗口**（如：找到字符串中所有字母异位词）：
   - **进**：`count[s[i] - 'a']++`
   - **算**：检查 `count` 数组是否符合要求。
   - **出**：`count[s[i - k + 1] - 'a']--`
2. **对象进入/移除成本高**（如：窗口内维护一个有序集合）：
   - 如果窗口内的逻辑很复杂（比如要动态求中位数），单纯的 `+=` 和 `-=` 搞不定。
   - 这时可能需要配合 `std::multiset` (C++) 或 `SortedList` (Python) 来进行删除和插入操作，复杂度会从 $O(N)$ 变为 $O(N \log K)$。

## 不定长滑动

**不定长滑动窗口**（Variable Length Sliding Window）是面试中出现频率最高的题型之一，比定长窗口稍微复杂一点。

它的核心不再是“一进一出”，而是 **“右边猛冲，左边看情况缩进”**。我们通常用 `left` 和 `right` 两个指针来维护窗口。

根据题目要求的不同（求**最长**还是**最短**），模板会有细微的区别。这是刷题时的**关键分水岭**。

------

### 场景一：求“最长”合法子数组

**典型题目**：无重复字符的最长子串、最大连续 1 的个数 III。

**逻辑口诀**：

1. `right` 主动向右移，数据进窗口。
2. **如果**窗口**不合法**（比如有重复字符了、0 的个数超标了），`left` 被迫向右移（缩圈），直到窗口再次合法。
3. 每次循环结束时，窗口一定合法，此时更新 `max_len`。

#### C++ 模板 (求最长)

C++

```c++
int maxSlidingWindow(vector<int>& nums) {
    int n = nums.size();
    int left = 0;
    int ans = 0;
    
    // 定义维护窗口状态的变量 (如 sum, count, map 等)
    int current_state = 0; 

    for (int right = 0; right < n; right++) {
        // 1. 【进】元素进入窗口，更新状态
        current_state += nums[right];

        // 2. 【缩】如果窗口不合法 (INVALID)，left 持续向右缩进
        while (/* 窗口不符合条件，例如 sum > target */) {
            // 移除 nums[left] 对状态的影响
            current_state -= nums[left];
            left++;
        }

        // 3. 【算】此时窗口一定是合法的，更新最大长度
        // 注意：长度是 right - left + 1
        ans = max(ans, right - left + 1);
    }
    return ans;
}
```

------

### 场景二：求“最短”合法子数组

**典型题目**：长度最小的子数组和（LeetCode 209）、最小覆盖子串（LeetCode 76）。

**逻辑口诀**：

1. `right` 主动向右移，数据进窗口。
2. **一旦**窗口**合法**（比如和 >= target 了），尝试更新 `min_len`。
3. 然后 `left` 尝试向右移（缩圈），看看能不能在保持合法的前提下让窗口更小。**循环执行此步**。

#### C++ 模板 (求最短)

C++

```c++
int minSlidingWindow(vector<int>& nums, int target) {
    int n = nums.size();
    int left = 0;
    // 初始化为最大整数，方便取 min
    int ans = INT_MAX; 
    int current_sum = 0;

    for (int right = 0; right < n; right++) {
        // 1. 【进】
        current_sum += nums[right];

        // 2. 【缩 & 算】只要窗口是合法的 (VALID)，就一直尝试缩小
        while (current_sum >= target) {
            // A. 在这里更新最短长度！因为现在的窗口是合法的
            ans = min(ans, right - left + 1);

            // B. 移出 nums[left]，尝试更短
            current_sum -= nums[left];
            left++;
        }
    }

    return ans == INT_MAX ? 0 : ans;
}
```



------

### ⚠️ 核心区别总结

| **特性**         | **求最长 (Longest)**             | **求最短 (Shortest)**          |
| ---------------- | -------------------------------- | ------------------------------ |
| **While 条件**   | 当窗口 **不合法** 时缩进         | 当窗口 **合法** 时缩进         |
| **更新答案位置** | While 循环 **外部** (保证合法后) | While 循环 **内部** (缩进之前) |
| **心态**         | "不得不缩" (为了合法)            | "尝试缩缩看" (为了更短)        |

### 实战演练

#### 1. (求最长) LeetCode 3. 无重复字符的最长子串

**题目**：给定一个字符串 `s` ，请你找出其中不含有重复字符的 **最长子串** 的长度。

C++

```c++
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        // AI Infra 选手习惯：用数组代替哈希表，性能起飞
        // vector<int> count(128, 0); 也可以
        int count[128] = {0}; 
        
        int left = 0;
        int ans = 0;
        
        for (int right = 0; right < s.size(); right++) {
            char c = s[right];
            // 1. 进
            count[c]++;
            
            // 2. 缩：如果当前字符出现次数 > 1，说明重复了，必须缩
            while (count[c] > 1) {
                char out = s[left];
                count[out]--;
                left++;
            }
            
            // 3. 算：此时窗口内绝对没有重复字符
            ans = max(ans, right - left + 1);
        }
        return ans;
    }
};
```

#### 2. (求最短) LeetCode 209. 长度最小的子数组

**题目**：找出该数组中满足其和 `≥ target` 的长度最小的子数组。

C++

```c++
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        int left = 0;
        int sum = 0;
        int ans = INT_MAX;
        
        for (int right = 0; right < nums.size(); right++) {
            // 1. 进
            sum += nums[right];
            
            // 2. 缩：只要和够大了，就记录答案，并尝试缩小窗口
            while (sum >= target) {
                ans = min(ans, right - left + 1); // 记录合法长度
                
                sum -= nums[left]; // 踢出左边
                left++;
            }
        }
        
        return ans == INT_MAX ? 0 : ans;
    }
};
```

# 求字数组个数

这是一个非常棒的归纳总结。在滑动窗口计数问题中，确实主要分为这两大类。它们的**窗口收缩逻辑 (`while`)** 和 **答案统计公式 (`ans += ...`)** 是截然不同的。

这里是两套标准模板，请务必记熟。

------

### 1. 越短越合法 (Shorter is Valid)

**典型场景**：乘积小于 K、和小于 K、最多包含 K 个不同字符。

**逻辑**：窗口过大时就会“爆掉”（不合法），必须收缩。

**核心**：统计 **以 `right` 结尾** 的合法子数组个数。

#### 📜 模板代码

C++

```c++
long long countSubarrays(vector<int>& nums, int k) {
    long long ans = 0;
    int left = 0;
    // 维护窗口状态的变量 (如 sum, prod, count map 等)
    long long state = 0; 

    for (int right = 0; right < nums.size(); right++) {
        // 1. 【进】加入 right
        // state += nums[right];

        // 2. 【缩】只要窗口“太大/不合法”，就收缩 left
        // 注意：这里条件是 invalid
        while ( /* state >= k (不合法) */ ) {
            // state -= nums[left];
            left++;
        }

        // 3. 【算】此时 [left, right] 是以 right 结尾的最大合法窗口
        // 那么 [left, right], [left+1, right] ... [right, right] 都是合法的
        // 个数 = 窗口长度
        ans += (right - left + 1);
    }
    return ans;
}
```

**对应题目**：

- LeetCode 713. 乘积小于 K 的子数组
- LeetCode 2302. 统计得分小于 K 的子数组数目

------

### 2. 越长越合法 (Longer is Valid)

**典型场景**：和至少为 K、至少包含 K 个不同字符、最大元素出现至少 K 次。

**逻辑**：窗口越大越容易满足条件。一旦满足条件，再往右扩依然满足（但我们通常想找最短的或统计所有）。

**核心**：统计 **以 `right` 结尾** 的合法子数组个数（但逻辑是反着推的）。

#### 📜 模板代码

C++

```c++
long long countSubarrays(vector<int>& nums, int k) {
    long long ans = 0;
    int left = 0;
    // 维护窗口状态的变量
    long long state = 0;

    for (int right = 0; right < nums.size(); right++) {
        // 1. 【进】加入 right
        // state += nums[right];

        // 2. 【缩】只要窗口“合法”，就尝试收缩 left (找极限)
        // 注意：这里条件是 valid，我们要把 left 缩到不合法为止
        while ( /* state >= k (合法) */ ) {
            // state -= nums[left];
            left++;
        }

        // 3. 【算】
        // 此时，while 循环结束，意味着 nums[left-1] ... nums[right] 是合法的，
        // 而 nums[left] ... nums[right] 刚变得不合法。
        // 这说明：以 right 结尾的子数组，左端点可以是 0, 1, ..., left-1
        // 这些左端点都让窗口足够长，满足条件。
        // 个数 = left
        ans += left;
    }
    return ans;
}
```

**逻辑详解**：

假如 `nums = [1, 2, 3, 4]`, `target = 6` (求和 $\ge 6$)。

- `right` 走到 `3` (对应值 4) 时，`[1, 2, 3, 4]` (sum=10) 合法。
- `while` 循环：
  - 扔掉 1，`[2, 3, 4]` (sum=9) 合法，`left` 变 1。
  - 扔掉 2，`[3, 4]` (sum=7) 合法，`left` 变 2。
  - 扔掉 3，`[4]` (sum=4) **不合法**，退出 `while`。
- 此时 `left = 2`。说明下标 `0` 和 `1` 作为起点的子数组 (`[1,2,3,4]` 和 `[2,3,4]`) 都是合法的。
- 所以 `ans += left`。

**对应题目**：

- LeetCode 2962. 统计最大元素出现至少 K 次的子数组
- LeetCode 1358. 包含所有三种字符的子字符串数目

### ⚡️ 极速对照表

| **特性**       | **越短越合法 (At Most / Less)** | **越长越合法 (At Least / More)**            |
| -------------- | ------------------------------- | ------------------------------------------- |
| **While 条件** | `while (不合法)`                | `while (合法)`                              |
| **收缩意图**   | 必须把窗口缩到合法为止          | 试图找到“不再合法”的那个临界点              |
| **统计公式**   | **`ans += right - left + 1;`**  | **`ans += left;`**                          |
| **物理含义**   | 当前窗口里的都是对的            | 刚才扔出去的那些 (`0` 到 `left-1`) 都是对的 |
